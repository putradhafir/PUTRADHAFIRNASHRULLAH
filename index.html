2. Class & Object

Pada source code ini, class Car memiliki dua atribut yaitu model dan color. Method start() dan stop() menggambarkan perilaku objek mobil. Class ini menunjukkan konsep dasar OOP yaitu pembuatan objek dari sebuah blueprint (class). 

class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

3. Constructor

Constructor digunakan untuk memberikan nilai awal saat objek dibuat. Dengan Car(this.model, this.color), setiap objek Car akan langsung memiliki nilai model dan color ketika diinisialisasi Ini meningkatkan efisiensi dan memastikan data tidak kosong 

class Car {
  String? model;
  String? color;

  Car(this.model, this.color);

  void start() {
    print('Car started!');
  }
}

4. Validasi Data

Validasi constructor merupakan proses pengecekan data yang diberikan saat pembuatan object agar sesuai dengan aturan atau ketentuan tertentu. Tujuan validasi adalah untuk mencegah terjadinya kesalahan data (invalid data) yang dapat menyebabkan bug atau ketidakkonsistenan dalam program. 

class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError('IPK harus 0.0 - 4.0');
    }
  }
}

5. Encapsulation

Untuk mengakses atau mengubah nilai atribut private, digunakan method khusus seperti getter dan setter. Getter berfungsi untuk mengambil nilai atribut, sedangkan setter digunakan untuk mengubah nilai atribut dengan kontrol tertentu. Dengan encapsulation, data menjadi lebih aman, terkontrol, dan terhindar dari manipulasi yang tidak diinginkan 

class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

6. Named Parameter Constructor

Penggunaan named parameter sangat berguna ketika sebuah function atau constructor memiliki banyak parameter. Selain meningkatkan keterbacaan (readability), named parameter juga mengurangi risiko kesalahan urutan pengisian nilai. Hal ini membuat kode lebih fleksibel dan profesional dalam pengembangannya. 

class Produk {
  final String id;
  final String nama;
  final double harga;
  final int stok;
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}
