Object Oriented Programming Laporan Object-Oriented Programming (OOP) adalah dokumen dokumentasi teknis yang menjelaskan desain, struktur, dan implementasi perangkat lunak berbasis objek. Laporan ini menjabarkan bagaimana entitas dunia nyata dimodelkan ke dalam kelas (class), objek, serta menerapkan empat pilar utama yaitu enkapsulasi, pewarisan (inheritance), polimorfisme, dan abstraksi untuk menciptakan kode yang modular, efisien, dan mudah dipelihara.

1. Class & Object Class merupakan blueprint atau cetakan yang digunakan untuk membuat sebuah object. Di dalam class terdapat atribut (data/variabel) dan method (fungsi) yang mendefinisikan perilaku suatu objek. Class berfungsi sebagai representasi konsep atau entitas dunia nyata ke dalam bentuk program. Misalnya, class Mahasiswa dapat memiliki atribut seperti nama, NIM, dan jurusan, serta method seperti tampilkanData(). Object adalah hasil instansiasi dari class. Object memiliki nilai nyata dari atribut yang telah didefinisikan dalam class dan dapat menjalankan method yang tersedia. Dengan menggunakan konsep class dan object, program menjadi lebih terstruktur, modular, dan mudah dikembangkan karena setiap entitas dipisahkan berdasarkan tanggung jawabnya.


class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }
}

3. Constructor Constructor adalah method khusus dalam sebuah class yang secara otomatis dipanggil ketika object dibuat. Fungsi utama constructor adalah untuk menginisialisasi nilai awal dari atribut yang dimiliki oleh class tersebut. Constructor memiliki nama yang sama dengan nama class dan tidak memiliki tipe data pengembalian (return type). 


class Car {
  String? model;
  String? color;

  Car(this.model, this.color);
}
Run in DartPad

4. Validasi Data Validasi constructor merupakan proses pengecekan data yang diberikan saat pembuatan object agar sesuai dengan aturan atau ketentuan tertentu. Tujuan validasi adalah untuk mencegah terjadinya kesalahan data (invalid data) yang dapat menyebabkan bug atau ketidakkonsistenan dalam program. 


class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk);
}
Run in DartPad

5. Encapsulation Encapsulation membatasi akses langsung ke atribut dengan menggunakan private variable serta menyediakan getter dan setter. 


class Person {
  String _name;
  int age;

  Person(this._name, this.age);
}
Run in DartPad

6. Named Parameter Named parameter adalah fitur dalam bahasa Dart yang memungkinkan parameter dipanggil berdasarkan nama, bukan hanya berdasarkan urutan posisi. Dengan cara ini, kode menjadi lebih jelas dan mudah dipahami karena setiap nilai yang diberikan langsung menunjukkan maksud penggunaannya. 


class Produk {
  final String id;
  final String nama;
  final double harga;

  Produk({required this.id, required this.nama, required this.harga});
}
