Pendahuluan

Laporan Object-Oriented Programming (OOP) adalah dokumen dokumentasi teknis yang menjelaskan desain, struktur, dan implementasi perangkat lunak berbasis objek. Laporan ini menjabarkan bagaimana entitas dunia nyata dimodelkan ke dalam kelas (class), objek, serta menerapkan empat pilar utama yaitu enkapsulasi, pewarisan (inheritance), polimorfisme, dan abstraksi untuk menciptakan kode yang modular, efisien, dan mudah dipelihara.

1. Class & Object

Class merupakan blueprint atau cetakan yang digunakan untuk membuat sebuah object. Di dalam class terdapat atribut (data/variabel) dan method (fungsi) yang mendefinisikan perilaku suatu objek. Class berfungsi sebagai representasi konsep atau entitas dunia nyata ke dalam bentuk program. Misalnya, class Mahasiswa dapat memiliki atribut seperti nama, NIM, dan jurusan, serta method seperti tampilkanData(). Object adalah hasil instansiasi dari class. Object memiliki nilai nyata dari atribut yang telah didefinisikan dalam class dan dapat menjalankan method yang tersedia. Dengan menggunakan konsep class dan object, program menjadi lebih terstruktur, modular, dan mudah dikembangkan karena setiap entitas dipisahkan berdasarkan tanggung jawabnya.

2. Constructor

Constructor adalah method khusus dalam sebuah class yang secara otomatis dipanggil ketika object dibuat. Fungsi utama constructor adalah untuk menginisialisasi nilai awal dari atribut yang dimiliki oleh class tersebut. Constructor memiliki nama yang sama dengan nama class dan tidak memiliki tipe data pengembalian (return type).

3. Validasi Constructor

Validasi constructor merupakan proses pengecekan data yang diberikan saat pembuatan object agar sesuai dengan aturan atau ketentuan tertentu. Tujuan validasi adalah untuk mencegah terjadinya kesalahan data (invalid data) yang dapat menyebabkan bug atau ketidakkonsistenan dalam program.

4. Encapsulation

Encapsulation membatasi akses langsung ke atribut dengan menggunakan private variable serta menyediakan getter dan setter.

5. Named Parameter

Named parameter membuat parameter lebih fleksibel dan mudah dibaca karena dipanggil berdasarkan nama.

6. Program Tambahan 1

Program tambahan pertama dibuat untuk memperluas pemahaman mengenai penerapan konsep OOP dalam situasi yang lebih kompleks. Pada bagian ini, class dirancang dengan atribut dan method tambahan yang menunjukkan bagaimana object dapat berinteraksi dan menjalankan logika tertentu sesuai kebutuhan program.

Melalui implementasi ini, dapat dilihat bagaimana konsep OOP membantu dalam memecah permasalahan menjadi bagian-bagian kecil yang terstruktur. Hal tersebut membuat program lebih mudah dikembangkan, diuji, dan dipelihara dalam jangka panjang.

7. Program Tambahan 2

Program tambahan kedua menunjukkan penerapan lanjutan dari konsep OOP dengan menggabungkan beberapa prinsip seperti encapsulation dan constructor dalam satu implementasi. Tujuannya adalah untuk memberikan gambaran nyata bagaimana OOP digunakan dalam pengembangan aplikasi yang lebih realistis.
