Pendahuluan

Laporan Object-Oriented Programming (OOP) adalah dokumen dokumentasi teknis yang menjelaskan desain, struktur, dan implementasi perangkat lunak berbasis objek. Laporan ini menjabarkan bagaimana entitas dunia nyata dimodelkan ke dalam kelas (class), objek, serta menerapkan empat pilar utama yaitu enkapsulasi, pewarisan (inheritance), polimorfisme, dan abstraksi untuk menciptakan kode yang modular, efisien, dan mudah dipelihara.

Melalui laporan ini, setiap konsep OOP dalam bahasa Dart akan dijelaskan dan diimplementasikan secara sistematis untuk menunjukkan bagaimana paradigma berorientasi objek membantu dalam pengembangan perangkat lunak yang terstruktur.

1. Class & Object

Class adalah blueprint atau cetakan untuk membuat object. Object merupakan hasil instansiasi dari class yang memiliki atribut dan method.

2. Constructor

Constructor adalah method khusus dalam class yang otomatis dipanggil saat object dibuat untuk menginisialisasi nilai atribut.

3. Validasi Constructor

Validasi memastikan data yang diberikan saat pembuatan object memenuhi aturan tertentu sehingga menjaga integritas data.

4. Encapsulation

Encapsulation membatasi akses langsung ke atribut dengan menggunakan private variable serta menyediakan getter dan setter.

5. Named Parameter

Named parameter membuat parameter lebih fleksibel dan mudah dibaca karena dipanggil berdasarkan nama.
