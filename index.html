1. PENGERTIAN OOP

Object Oriented Programming (OOP) atau Pemrograman Berorientasi Objek adalah paradigma pemrograman yang menggunakan konsep "objek" untuk merepresentasikan data dan perilaku dalam program.

Dalam OOP, segala sesuatu diibaratkan sebagai objek yang memiliki dua karakteristik utama:

  • Atribut (Data/Property): Informasi yang dimiliki oleh objek (contoh: nama, warna, ukuran)
  • Method (Perilaku/Function): Tindakan yang dapat dilakukan oleh objek (contoh: berjalan, berbicara, menghitung)

OOP dirancang untuk meniru cara manusia memandang dunia nyata. Misalnya, ketika kita melihat mobil, kita mengenali bahwa mobil memiliki atribut (warna, merek, model) dan perilaku (menyalakan mesin, mengerem, berbelok). OOP menerapkan konsep yang sama ke dalam kode program.

Keuntungan Utama OOP:

  • Modularitas: Program dibagi menjadi bagian-bagian kecil yang independen dan mudah dikelola
  • Reusability: Kode dapat digunakan kembali tanpa perlu ditulis ulang dari awal
  • Maintainability: Lebih mudah diperbaiki dan diupdate karena strukturnya yang terorganisir
  • Scalability: Mudah dikembangkan seiring bertambahnya kebutuhan sistem

Empat Pilar Utama OOP:

  • Encapsulation: Membungkus data dan method dalam satu unit (class) dan membatasi akses langsung ke data
  • Abstraction: Menyembunyikan detail kompleks dan hanya menampilkan fitur yang penting
  • Inheritance: Kemampuan class untuk mewarisi sifat dan perilaku dari class lain
  • Polymorphism: Kemampuan objek untuk mengambil banyak bentuk (bisa merespon method yang sama dengan cara berbeda)

Dalam bahasa pemrograman Dart, OOP didukung secara native. Dart memungkinkan kita membuat class, object, constructor, method, dan menerapkan keempat pilar OOP dengan mudah. Konsep ini menjadi fondasi utama dalam pengembangan aplikasi modern, termasuk framework Flutter yang menggunakan Dart sebagai bahasa utamanya.

Pada source code ini, class Car memiliki dua atribut yaitu model dan color. Method start() dan stop() menggambarkan perilaku objek mobil. Class ini menunjukkan konsep dasar OOP yaitu pembuatan objek dari sebuah blueprint (class).

2. CLASS DAN OBJECT


class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

Constructor digunakan untuk memberikan nilai awal saat objek dibuat. Dengan Car(this.model, this.color), setiap objek Car akan langsung memiliki nilai model dan color ketika diinisialisasi. Ini meningkatkan efisiensi dan memastikan data tidak kosong.

3. CONSTRUCTOR


class Car {
  String? model;
  String? color;

  Car(this.model, this.color);

  void start() {
    print('Car started!');
  }
}

4. VALIDASI DATA

Atribut name diawali dengan underscore () yang menandakan bahwa variabel tersebut bersifat private. Akses dilakukan melalui method setName() dan getName(). Konsep ini disebut Encapsulation, yaitu membatasi akses langsung ke data. 


class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk) {
    if (ipk < 0.0 || ipk > 4.0) {
      throw ArgumentError('IPK harus 0.0 - 4.0');
    }
  }
}

5. ENCAPSULATION


class Person {
  String _name;
  int age;

  Person(this._name, this.age);

  void setName(String name) {
    _name = name;
  }

  String getName() {
    return _name;
  }
}

Constructor dengan named parameter membuat kode lebih jelas dan fleksibel. Kata kunci required memastikan nilai wajib diisi, sementara nilai default seperti stok = 0 dan kategori = 'Umum' memberikan keamanan tambahan jika tidak diinput.

6. NAMED PARAMETER


class Produk {
  final String id;
  final String nama;
  final double harga;
  final int stok;
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}
