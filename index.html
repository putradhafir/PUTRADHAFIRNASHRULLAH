1. Pengertian Object Oriented Programming (OOP)

Object Oriented Programming (OOP) adalah paradigma pemrograman yang berorientasi pada objek. Dalam OOP, program dibangun berdasarkan objek yang merupakan representasi dari entitas nyata maupun abstrak. Setiap objek dibuat dari sebuah class yang berfungsi sebagai cetak biru (blueprint).

Tujuan utama OOP adalah membuat program lebih terstruktur, mudah dikembangkan, serta meningkatkan keamanan dan penggunaan ulang kode. Dalam bahasa Dart, konsep OOP sangat penting karena Dart dirancang sebagai bahasa yang sepenuhnya mendukung pemrograman berbasis objek.

2. Class dan Object

Class adalah cetak biru atau template untuk membuat objek. Di dalam class terdapat atribut (data) dan method (fungsi) yang mendefinisikan perilaku suatu objek. Object adalah hasil instansiasi dari class.

Dengan menggunakan class dan object, program menjadi lebih terorganisir karena setiap data dan fungsi dikelompokkan sesuai fungsinya. 


class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }
}

3. Constructor

Constructor adalah method khusus yang digunakan untuk menginisialisasi nilai awal suatu objek saat objek dibuat. Constructor memiliki nama yang sama dengan class.

Fungsi constructor adalah memberikan nilai awal pada atribut sehingga object siap digunakan. 


class Car {
  String? model;
  String? color;

  Car(this.model, this.color);
}

4. Validasi Data dalam Class

Validasi data adalah proses pengecekan terhadap data yang dimasukkan agar sesuai dengan aturan yang telah ditentukan. Validasi biasanya dilakukan di dalam constructor atau method setter.

Tujuan validasi adalah mencegah data tidak valid masuk ke dalam object dan menjaga konsistensi program. 


class Mahasiswa {
  String nama;
  String nim;
  double ipk;

  Mahasiswa(this.nama, this.nim, this.ipk);
}

5. Encapsulation

Encapsulation adalah konsep OOP yang membatasi akses langsung ke atribut suatu class. Data biasanya dibuat private agar tidak dapat diakses secara langsung dari luar class.

Akses terhadap data dilakukan melalui method getter dan setter sehingga keamanan dan kontrol data lebih terjaga. 


class Person {
  String _name;
  int age;

  Person(this._name, this.age);
}

6. Constructor dengan Named Parameter

Named parameter memungkinkan pengisian parameter dengan menyebutkan nama variabelnya secara langsung saat pemanggilan constructor. Hal ini membuat kode lebih mudah dibaca dan fleksibel.

Dengan menggunakan kata kunci required, kita dapat memastikan parameter wajib diisi. 


class Produk {
  final String id;
  final String nama;
  final double harga;

  Produk({required this.id, required this.nama, required this.harga});
}

7. Kesimpulan

Object Oriented Programming (OOP) merupakan paradigma pemrograman yang membantu dalam membangun aplikasi yang terstruktur, aman, dan mudah dikembangkan.

Konsep seperti Class, Object, Constructor, Validasi Data, Encapsulation, dan Named Parameter membantu programmer dalam mengelola kompleksitas program serta meningkatkan kualitas kode. Dalam Dart, penerapan OOP sangat penting karena bahasa ini berbasis objek dan banyak digunakan dalam pengembangan aplikasi modern seperti Flutter.
